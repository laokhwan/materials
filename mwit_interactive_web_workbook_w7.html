เริ่มต้น: เว็บแอปหนึ่งตัวทำงานอย่างไร?

เป้าหมายของบทนี้: จากเดิมที่เราเขียนเว็บให้แสดงข้อมูลได้ วันนี้เราจะทำให้เว็บรับข้อมูลจากผู้ใช้ คำนวณ แสดงผล และบันทึกข้อมูลไว้ในเครื่องของผู้ใช้ด้วย localStorage

ภาพรวมการทำงาน

Input       ผู้ใช้กรอกข้อมูล เช่น ชื่อ คะแนน จำนวนชั่วโมง
Process     JavaScript อ่านค่า คำนวณ และตัดสินใจ
Output      แสดงผลลัพธ์กลับไปยังหน้าเว็บ
Storage     บันทึกข้อมูลไว้ใน Browser ด้วย Local Storage

Input

ช่องรับข้อมูล เช่น input, select, radio, checkbox, textarea

Process

การประมวลผล เช่น Number(), if/else, การคำนวณ และการสร้างคำแนะนำ

Storage

การบันทึกข้อมูลไว้ใน Browser ด้วย localStorage เพื่อให้ข้อมูลยังอยู่หลัง Refresh

Lab 5: บันทึกข้อมูลด้วย Local Storage

localStorage ใช้บันทึกข้อมูลไว้ใน Browser ข้อมูลจะยังอยู่แม้ Refresh หน้าเว็บหรือปิดแล้วเปิดใหม่ แต่ข้อมูลนี้ไม่ได้ส่งไปให้เพื่อนหรือเก็บบน Cloud

คำสั่งพื้นฐาน

คำสั่งความหมาย
localStorage.setItem(key, value)บันทึกข้อมูล
localStorage.getItem(key)อ่านข้อมูล
localStorage.removeItem(key)ลบข้อมูล

ภารกิจ: สร้างเว็บแอปที่รับข้อมูลแผนการอ่านหนังสือ คำนวณเวลาเฉลี่ยต่อบท แสดงคำแนะนำ และบันทึกข้อมูลล่าสุดไว้ใน Local Storage

ข้อมูลที่ต้องรับเข้า

  • ชื่อนักเรียน
  • วิชาที่ต้องการอ่าน
  • จำนวนบทที่ต้องอ่าน
  • จำนวนชั่วโมงที่มี
  • ระดับความยากของวิชา
  • เป้าหมายการอ่าน
  • วิธีอ่านที่วางแผนจะใช้
  • บันทึกเพิ่มเติม

ฟังก์ชันที่ต้องทำได้

  • อ่านค่าจาก input ได้ถูกต้อง
  • ตรวจสอบข้อมูลก่อนคำนวณ
  • คำนวณเวลาเฉลี่ยต่อบท
  • แสดงคำแนะนำ
  • Save / Load / Clear ด้วย Local Storage

Checkoff และ Reflection

Checkoff

  • อธิบายได้ว่า input แต่ละชนิดเหมาะกับข้อมูลแบบใด
  • อ่านค่าจาก input ด้วย .value ได้
  • แปลงข้อมูลเป็นตัวเลขด้วย Number() ได้
  • ใช้ if / else เพื่อสร้างคำแนะนำได้
  • แสดงผลบนหน้าเว็บด้วย innerHTML หรือ textContent ได้
  • ใช้ localStorage.setItem(), getItem(), removeItem() ได้
  • อธิบายได้ว่า JSON.stringify() และ JSON.parse() ใช้ทำอะไร

Reflection

  1. วันนี้เว็บแอปของคุณรับข้อมูลอะไรจากผู้ใช้บ้าง?
  2. ข้อมูลใดต้องแปลงเป็นตัวเลขก่อนนำไปคำนวณ?
  3. Local Storage แตกต่างจากตัวแปร JavaScript อย่างไร?
  4. Local Storage เหมาะกับข้อมูลประเภทใด และไม่เหมาะกับข้อมูลประเภทใด?
  5. ถ้าต้องการให้เพื่อนหลายคนเห็นข้อมูลเดียวกัน Local Storage เพียงพอหรือไม่ เพราะอะไร?
  6. ถ้าใช้ AI ช่วยเขียนโค้ด คุณตรวจสอบความถูกต้องอย่างไร?
